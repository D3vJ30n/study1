# 1. 객체 리터럴

## 장점

### 1. 단순하고 직관적<br>
● `객체 리터럴은 {} 구문을 사용하여 매우 간단`하게 객체를 정의할 수 있어 코드가 직관적이고 읽기 쉽다.<br>

## 예시<br>
```js
const person = {
    name: "John",
    age: 30
};
```
### 2. 빠른 초기화<br>
● 새로운 객체를 생성할 때 필요한 데이터를 한 번에 설정할 수 있어 초기화가 빠르고 간단.<br>

### 3. 구조적 유연성<br>
● 객체 리터럴은 중첩된 객체를 포함할 수 있으며, 복잡한 데이터를 표현하는 데 유용.<br>

## 단점
### 1. 재사용성 부족<br>
● 동일한 구조의 객체를 여러 번 생성해야 할 때는 코드가 중복될 수 있다.<br>

### 2. 상속 지원 부족<br>
● 객체 리터럴은 상속을 직접적으로 지원하지 않기 때문에 상속이 필요한 경우에는 비효율적.<br>

# 2. 생성자 함수

## 장점

### 1. 프로토타입 상속<br>
● 생성자 함수는 프로토타입을 통해 상속을 구현할 수 있어, 공통 메서드를 여러 인스턴스 간에 공유할 수 있다.<br>

## 예시<br>
```js
function Person(name, age) {
    this.name = name;
    this.age = age;
}

Person.prototype.greet = function() {
    console.log(`Hello, my name is ${this.name}`);
};
```
### 2. 동적 객체 생성<br>
● 생성자 함수를 통해 동적으로 객체를 생성할 수 있으며, 필요에 따라 다양한 파라미터를 받을 수 있다.<br>

## 단점

### 1. 문법적 복잡성<br>
● 생성자 함수와 프로토타입 개념은 객체 리터럴에 비해 상대적으로 복잡하며, 새로운 개발자에게 혼란을 줄 수 있다.<br>

### 2. `new` 키워드 필요<br>
● 생성자 함수는 new 키워드를 필요로 하며, 이를 빼먹을 경우 코드가 의도치 않게 동작할 수 있다.<br>

# 3. 클래스

## 장점

### 1. 문법적 편의성<br>
● ES6에서 도입된 클래스 문법은 Java나 C++와 같은 전통적인 객체 지향 언어의 문법과 유사하여, 객체 지향 프로그래밍에 익숙한 개발자에게 친숙.

## 예시<br>
```js
class Person {
    constructor(name, age) {
        this.name = name;
        this.age = age;
    }
    
    greet() {
        console.log(`Hello, my name is ${this.name}`);
    }
}
```

### 2. 상속과 캡슐화<br>
● 클래스는 상속을 통해 기존 클래스를 확장할 수 있으며, constructor와 메서드를 통해 캡슐화를 구현할 수 있다.

### 3. 정적 메서드 및 필드<br>
● 클래스는 정적 메서드와 정적 필드를 지원하여, 인스턴스화 없이도 메서드 호출이나 속성 접근이 가능.

## 단점

### 1. 실행 퍼포먼스<br>
● 클래스 문법은 생성자 함수와 비슷한 방식으로 동작하지만, 클래스 자체는 일부 경우에 퍼포먼스 오버헤드가 발생할 수 있다.

### 2. 호환성<br>
● 클래스 문법은 ES6 이상에서만 지원되므로, 구형 브라우저에서는 사용하기 위해 트랜스파일러(babel 등)를 사용해야 한다.

## 결론 및 인사이트
● 단순한 데이터 구조: 단순한 객체는 객체 리터럴을 사용하여 정의하는 것이 가장 효율적입니다. 이는 코드가 간결하고 직관적이며, 작성 및 유지보수가 용이하기 때문이다.<br>
<br>
● 재사용 가능한 객체 생성: 여러 인스턴스가 필요한 경우 생성자 함수나 클래스를 사용합니다. 이 두 가지 방법은 재사용성과 확장성을 제공하며, 코드의 중복을 줄여준다.<br>
<br>
● 복잡한 객체 지향 프로그래밍: 상속과 같은 복잡한 객체 지향 기능이 필요하다면 클래스 문법을 사용하는 것이 좋습니다. 이는 더 명확한 문법과 구조를 제공하여, 코드의 가독성과 유지보수성을 향상시킨다.<br>

개발자는 각 방법의 장단점을 이해하고, 프로젝트의 요구사항과 상황에 맞게 적절한 객체 생성 방식을 선택하는 것이 중요하다.
