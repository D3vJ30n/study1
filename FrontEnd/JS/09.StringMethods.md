# 1. 문자열 메서드
## 1. 정의<br>
자바스크립트의 문자열 메서드는 문자열을 조작하고 처리하는 데 사용되는 내장 함수들이다. 주요 문자열 메서드와 그 정의는 다음과 같다.

1. split(): 문자열을 지정된 구분자를 기준으로 나누어 배열로 반환.
   예: "Hello,World".split(",") // ["Hello", "World"]

2. length: 문자열의 길이를 반환.
   예: "Hello".length // 5

3. toLowerCase(): 문자열을 모두 소문자로 변환.
   예: "Hello".toLowerCase() // "hello"

4. toUpperCase(): 문자열을 모두 대문자로 변환.
   예: "Hello".toUpperCase() // "HELLO"

5. trim(): 문자열의 양 끝 공백을 제거.
   예: "  Hello  ".trim() // "Hello"

6. slice(startIndex, endIndex): 문자열의 특정 부분을 추출.
   예: "Hello".slice(1, 4) // "ell"

7. replace(searchValue, replaceValue): 문자열 내의 특정 값을 다른 값으로 대체.
   예: "Hello".replace("H", "J") // "Jello"

8. charAt(index): 지정된 인덱스의 문자를 반환.
   예: "Hello".charAt(1) // "e"

이러한 메서드들은 문자열을 다양한 방식으로 조작하고 분석하는 데 사용된다. 문자열은 불변(immutable)이므로, 이 메서드들은 원본 문자열을 변경하지 않고 새로운 문자열을 반환한다는 점을 기억해야 한다. 이 메서드들을 활용하면 문자열 처리 작업을 효율적으로 수행할 수 있다.

## 2. 사용사례<br>
1. 문자열 검색 및 추출<br>
indexOf(), includes(), startsWith(), endsWith() 메서드를 사용하여 특정 문자열이나 패턴을 찾을 수 있다. 예를 들어, 파일명에서 특정 날짜나 작성자를 찾는 데 활용할 수 있다.

```JS
function findMarchFiles(arr) {
  return arr.filter(file => file.startsWith('202103'));
}
```

2. 문자열 분리 및 결합<br>
split() 메서드를 사용하여 문자열을 배열로 분리하거나, join() 메서드로 배열을 다시 문자열로 결합할 수 있습니다
```JS
const fullName = "John Doe";
const [firstName, lastName] = fullName.split(' ');
```

3. 문자열 변형<br>
toUpperCase(), toLowerCase(), trim(), replace() 등의 메서드를 사용하여 문자열을 변형할 수 있다.
```JS
const cleanedInput = userInput.trim().toLowerCase();
```

4. 문자열 추출<br>
substring(), slice() 메서드를 사용하여 문자열의 일부를 추출할 수 있다.
```JS
const year = fileName.substring(0, 4);
```

5. 문자열 패딩<br>
padStart(), padEnd() 메서드를 사용하여 문자열의 길이를 일정하게 맞출 수 있다.
```JS
const formattedNumber = number.toString().padStart(3, '0');
```

6. 정규표현식과 함께 사용<br>
match(), replace(), search() 메서드는 정규표현식과 함께 사용하여 복잡한 문자열 처리 작업을 수행할 수 있다.
```JS
const hasNumber = /\d/.test(password);
```

7. 템플릿 리터럴과 함께 사용<br>
백틱(`)을 사용한 템플릿 리터럴과 함께 문자열 메서드를 조합하여 동적인 문자열을 생성할 수 있다.
```JS
const greeting = `Hello, ${name.toUpperCase()}!`;
```


## 3. 인사이트

✅ 기본기의 중요성<br>
문자열 메서드는 프로그래밍의 기본 중의 기본이다. 이를 깊이 이해하고 능숙하게 다룰 수 있다면, 복잡한 문제도 효율적으로 해결할 수 있다. 기본기를 탄탄히 다지는 것이 중요하다는 것을 깨달았다.<br>
<br>
✅ 문서화의 가치<br>
MDN 웹 문서와 같은 공식 문서를 통해 각 메서드의 정확한 사용법과 예제를 학습했다. 좋은 문서화의 가치를 인식하게 되었고, 앞으로 제가 작성하는 코드도 잘 문서화하려고 노력해야겠다고 생각했다.<br>
<br>
✅ 실전 적용의 중요성<br>
단순히 메서드를 암기하는 것보다 실제 프로젝트에 적용해보는 것이 훨씬 효과적인 학습 방법이라는 것을 깨달았다. 작은 프로젝트라도 직접 만들어보며 이 메서드들을 활용해보는 것이 중요하다.<br>
<br>
✅ 성능 고려의 필요성<br>
같은 결과를 얻을 수 있는 여러 방법 중에서 어떤 것이 더 효율적인지 고민하게 되었다. 예를 들어, 정규표현식을 사용하는 것과 단순 문자열 메서드를 사용하는 것 중 어떤 것이 더 성능이 좋은지 등을 고려하게 되었다.<br>
<br>
✅ 지속적인 학습의 중요성<br>
자바스크립트는 계속 발전하고 있어 새로운 메서드가 추가되고 있다. 이를 통해 기술 발전 속도가 빠르다는 것을 실감했고, 지속적인 학습의 필요성을 느꼈다.<br>
<br>
✅ 코드 가독성 향상<br>
적절한 문자열 메서드를 사용하면 복잡한 로직을 간결하고 읽기 쉬운 코드로 표현할 수 있다는 것을 알게 되었다. 이는 협업 시 매우 중요한 요소라고 생각한다.<br>
<br>
✅ 크로스 브라우저 호환성 고려<br>
일부 최신 메서드는 구형 브라우저에서 지원되지 않을 수 있다는 것을 알게 되었다. 실제 프로젝트에서는 타겟 사용자의 환경을 고려해야 한다는 점을 배웠다.<br>
<br>
✅ 문제 해결 능력 향상<br>
다양한 문자열 처리 문제를 해결하면서 논리적 사고력과 문제 해결 능력이 향상되었음을 느꼈다. 이는 개발자로서 성장하는 데 큰 도움이 될 것 같다.<br>
<br>
✅ 코드 최적화의 중요성<br>
같은 결과를 얻을 수 있는 여러 방법 중에서 가장 효율적인 방법을 선택하는 것의 중요성을 깨달았다. 이는 대규모 애플리케이션 개발 시 특히 중요할 것 같다.<br>
<br>
✅ 창의적 문제 해결<br>
여러 문자열 메서드를 조합하여 복잡한 문제를 해결하는 과정에서 창의적 사고의 중요성을 느꼈다. 프로그래밍은 단순한 코딩 이상의 창의적인 활동이라는 것을 깨달았다.<br>
